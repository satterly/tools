Jan 20 08:20 UTC
Hey there guys! I'm trying something and couldn't figure it out yet. I'd like to use alerta with keycloak using OIDC. I can assign roles and see them in alerta but how can I assign permissions from keycloak? Is there any best practice that you'd like to share? I saw that alerta app passes the "scopes" variable and calls them in api. Any help is much appreciated :)
Burak Köseoğlu
@burakksglu
Jan 20 08:49 UTC
I also know that Alerta uses RBAC(Role Based Access Control) but being able to send roles with set permissions from keycloak is my goal.
Nick Satterly
@satterly
Jan 20 09:34 UTC
If using OIDC you make use of a "roles" claim in the JWT. ie. you assign users to roles defined in Keycloak and then in Alerta you map Alerta permissions (ie. scopes like "read:alerts") to those roles.
Note: you can whitelist a list of roles that are allowed to login to Alerta using ALLOWED_OIDC_ROLES server setting
