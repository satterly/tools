Nick Satterly
@satterly
Jan 24 08:13 UTC
The only bulk endpoint that uses celery at the moment is the /api/_bulk/alerts/action endpoint. All of the others are not asynchronous. They need to be converted to use celery. So yes, it works for you with the /status endpoint without celery at the moment but that is not the intention. I would caution against using it as the API will change. ie. in future it will only return a task ID that tells you the bulk request has been queued for action. Then you poll /api/_bulk/task/<task_id> to get the celery status.
