Nick Satterly
@satterly
Feb 26 07:28 UTC
@mertbakir are you sure the API endpoint path is correct? a 405 Method Not Allowed is usually returned when you have the API endpoint wrong.
This worked for me …
$  cat alerta405.py
#!python3

import requests

apiEndpoint = "https://alerta-api.herokuapp.com/alert"
APIKey = "keyhere"

data = {
      "attributes": {
        "region": "EU"
      },
      "correlate": [
        "HttpServerError",
        "HttpServerOK"
      ],
      "environment": "DEV",
      "event": "HttpServerError",
      "group": "Web",
      "origin": "curl",
      "resource": "web01",
      "service": [
        "example.com"
      ],
      "severity": "major",
      "tags": [
        "dc1"
      ],
      "text": "Site is down.",
      "type": "exceptionAlert",
      "value": "Bad Gateway (501)"
    }

session = requests.session()
response = session.post(
            apiEndpoint,
            json = data,
            headers={
            'Content-Type': 'application/json',
            'Authorization': 'Key ' + APIKey,
            }
        )


print("Status Code:", response.status_code)
print(response.text)
Note if you’re using API keys you need to prepend “Key” the header value.
And if you’re writing this in python you should just use the supported python SDK https://pypi.org/project/alerta/
