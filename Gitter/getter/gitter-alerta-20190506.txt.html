The postgres python package is causing lots of pain since it went to version 2.8 and they don’t distribute binaries anymore. you have to build from source everytime you install which requires loads of dev packages and bloat — and pain.
I’m seriously thinking about dropping Postgres from the setup.py and make it an optional install for those that what to use postgres instead of mongo.
It would mean anyone wanting to get going quickly can do that (using mongo), and those that want a more production ready install can consider the pros and cons of mongo vs postgres for their environment and make the extra effort to use postgres, if that’s what they decide.
