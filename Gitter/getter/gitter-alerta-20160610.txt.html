@b1
@bl1nk if you just want to modify an alert (based on any attribute) then do it in the pre-receive hook.
then you don’t need to do the db.save_correlated() at all.
it’s only if you want to modify the alert in the post-receive hook following a request to an external system (for example, to update the alert with a pagerDuty notification id or something ) that you then need to do the additional work of updating the alert using db.save_correlated()
this example shows you how to modify an alert in the pre-receive hook (before it is automatically saved to the database) …
class EnhanceAlert(PluginBase):

    def pre_receive(self, alert):
        if 'TPS reports' in alert.text:
            alert.attributes['customer'] = 'Initech'
        elif 'nexus' in alert.text:
            alert.attributes['customer'] = 'Tyrell Corp.'
        elif 'green wafer' in alert.text:
            alert.attributes['customer'] = 'Soylent Corp.'
        elif 'Skynet' in alert.text:
            alert.attributes['customer'] = 'Cyberdyne Systems'
        else:
            alert.attributes['customer'] = 'ACME Corp.'

        alert.attributes['runBookUrl'] = 'http://www.mywiki.org/RunBook/%s' % alert.event.replace(' ', '-')

        return alert

    def post_receive(self, alert):
        return

    def status_change(self, alert, status, text):
        return
