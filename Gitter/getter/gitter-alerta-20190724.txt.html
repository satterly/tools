Ok. So there’s a bug with this setup that is fixed by alerta/alerta#1031 and alerta/alerta#1033
Until #1033 is merged you’ll need to add ‘normal’ to your severity map.
this is because ‘normal’ is hardcoded as the “OK” status. but you don’t have ‘normal’ in your list of valid severities.

Jul 24 2019 23:40 UTC

@satterly I have another issue this time, Blackouts. Blackout is not working properly.
I set the blackout period based on environment and resource. When alert.status == alerting, blackout doesnt block the alert from getting processed by plugins. But when alert.status == ok or normal, blackout blocks it.

I am using postgres as backend db. Did some digging... found that def is_blackout_period(self, alert): in postgres/base.py is returning different boolean values. Even though blackout is enabled, for an incoming alert is in alerting state, it returns False and when alert in is ok state it return True

As a result, blackout doesnt work when alert is alerting but works when alert is cleared or ok.
