Nick Satterly
@satterly
May 15 2018 11:17 UTC
You need to set the ALERTA_ENDPOINT environment variable to the actual URL of your Alerta API. Don’t assume that “localhost” will work. 
$ vi /etc/default/snmptrapd

export MIBS=+ALL
TRAPDRUN=yes
export ALERTA_ENDPOINT="http://localhost:8080"
The handler.py doesn’t call the transform plugin. The transform plugin is run by the Alerta API after it receives the alert. Just like any other plugin.
@Ziomalon take at look at this tutorial http://docs.alerta.io/en/latest/gettingstarted/tutorial-3-plugins.html#step-3-route-alerts-to-plugins it explains how to do exactly what you’re asking for.

Nick Satterly
@satterly
May 15 2018 12:31 UTC
yes. the idea is to use a “transform” plugin to map the different SNMP varbinds to useful alerta attributes. every trap definition is different so there was no other way to do this so it works for everyone.
when the snmptrap handler creates an alert it does a HTTP POST request to the URL formed by taking the ALERTA_ENDPOINT URL and adding “/alert” on the end. this is a standard approach for implementing a REST API. see http://docs.alerta.io/en/latest/api/reference.html#create-an-alert
