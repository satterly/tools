Steven Bambling

Mar 04 11:50 UTC

@satterly
I have my current testing setup with two MLM nodes (MLM1/MLM2) and a single MOM. The MLM nodes are set to forward both action and
alerts with ['*'] and the MOM is set to only forward actions ['actions'] to prevent passing alert from MLM1 -> MOM
-> MLM2

Hit a little bit of a bug where actions fail with more then a single MLM because of the current logic that just does a simple loop though all of the
configured forward destinations. https://github.com/alerta/alerta/blob/alert-forwarder-plugin/alerta/plugins/forwarder.py#L61

This is causing the plugin to fail as its trying to perform an action against an alert that doesn't exist on one or more of
the systems.

For example:

An alert is forwarded from MLM2 -> MOM. The delete actions is trigged on the MOM and it then attempts to delete the
alerts from all entries in its FWD_DESTINATIONS so both MLM1 and MLM2. Yet MLM1 does NOT have that alert and the
delete action fails.

Note actions could sometimes work depending on the order of the remote MLM in the FWD_DESTINATIONS dict.

A possible solution could be something simple like an additional attribute with the origin added to events when they are forwarded.
This would only add the new attribute to forwarded and events on the MOM(s). Then possible logic could be if the
new attribute exists only delete from FWD_DESTINATIONS matching that remote. If the new attibute doesn't exist then forwarded
to every destination in FWD_DESTINATIONS. This should also allow MLM to forward events to more then a single MOM
for failover as well.

environment variable values are passed as comma-separated variables. Eg. not ALLOWED_ENVIRONMENTS="['pr', 'ap', 'ts','dv’]” but ALLOWED_ENVIRONMENTS=pr,ap,ts,dv
same for PLUGINS ie. PLUGINS=reject,msteams
if you search for “PLUGINS” in this README you’ll see an example docker-compose file that sets PLUGINS like that… https://hub.docker.com/r/alerta/alerta-web
