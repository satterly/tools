Nick Satterly

Dec 16 2018 21:12 UTC
One possible use case is if operators want to watch alerts then you must login first so that Alerta knows who has clicked the “watch” button. However, perhaps your Alerta API is on a protected network, not exposed to the internet, so there is no security reason to enforce authentication.
So by setting “AUTH_REQUIRED=False” you could allow anyone to do anything on alerta but if operators want to watch an alert they need to login first.
Nick Satterly
@satterly
Dec 16 2018 21:18 UTC
Another possible use case could be to ensure that all systems that send alerts to Alerta use an API key so that you can revoke it if you don’t want alerts from that system any more. You can still have “AUTH_REQUIRED=False” but if an invalid API is presented as credential it will be denied.
