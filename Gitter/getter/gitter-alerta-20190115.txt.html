yes. as i mentioned before, do not modify any config inside a docker container. it will be lost when the container is terminated. you should create the config file on the docker host (or elsewhere) and mount the config into the container at boot time.
from the docker readme… (but this is not unqiue to Alerta, all docker containers work in a similar way) … 
Configuration Files
To set configuration settings not supported by environment variables use configuration files instead. For example:

$ docker run -v $PWD/config/alertad.conf:/app/alertad.conf \
  -v $PWD/config/config.js:/app/config.js \
  -p <port>:8080 alerta/alerta-web
i usually find it easier to use docker compose anyway. it handles a lot of this for you. see this complete working example for a prometheus stack that forwards alerts to Alerta … https://github.com/alerta/docker-alerta/blob/master/examples/prometheus/docker-compose.yml
or this ldap example which uses a alertad.conf file in the config subdir and mounts it at run time … https://github.com/alerta/docker-alerta/tree/master/examples/ldap/config
