@satterly could you give me advice about using Alerta in my environment? Let's imagine that there is a service with srv1 resource field. This service contains two internal jobs and these jobs export some metrics. Also, there is an alert rule with the name JobFailed which generates alerts like this:

{name="job1",event="JobFailed",severity="critical",resource="srv1",environment="production", ...other necessary fields...}
{name="job2",event="JobFailed",severity="critical",resource="srv1",environment="production", ...other necessary fields...}

The difference between these alerts is only in the name field.

If I send these alerts to Alerta, it will consider them as duplicate alerts and I will have only one alert in Alerta as a result. It is not what I want - I want to see two alerts in Alerta if I have two broken jobs in one service.

In reality, I have many services and they contain many internal jobs but there is only one alert rule to notify about the bad status of them.
What can I do to differentiate alerts for each job in each service?
