Hello again. Is it possible to filter by an custom attribute in the Web UI?
Nick Satterly
@satterly
Jul 03 2019 22:38 UTC
@cisco92 Yes, you can do this ...
ie. ?q=attributes.region:EU
You can also query for alerts that have region attribute like this …
?q=_exists_:region
documented here .. https://docs.alerta.io/en/latest/api/query-syntax.html#field-names
Nick Satterly
@satterly
Jul 03 2019 22:44 UTC
@kklemen yes, use dynamic “per-alert” config for a plugin. That is, the routing rule can generate the config required by any called plugins based on the what’s in the alert. Background here … alerta/alerta#746 Fix here … alerta/alerta#944
@fshaikh-eg Yes, this is how Grafana and Prometheus Alertmanager work. See their web hooks for working examples … https://github.com/alerta/alerta/blob/master/alerta/webhooks/grafana.py and https://github.com/alerta/alerta/blob/master/alerta/webhooks/prometheus.py
You would write your own webhook plugin that worked the same way. See https://github.com/alerta/alerta-contrib/tree/master/webhooks for examples of custom webhook plugins.
Nick Satterly
@satterly
Jul 03 2019 22:50 UTC
@pkvel01 you might see the error in the Alerta API log but you should definitely see the POST body response in the Alertmanager log. I don’t know if you need to enable debug in Alertmanager first though.
