No you can’t configure the ack color. See this PR for list of things that can be changed … alerta/alerta-webui#193
@mgiammarco watch just allows a logged in user to keep track of alerts they are interested in. If a user watches an alert then they can hit the toggle button in the top nav bar and only see those watched alerts.
Nick Satterly
@satterly
Oct 01 2019 21:08 UTC
ack means that alert is being worked on, by someone. usually the person who acks it. if you have “notification sounds” enabled then the sound will stop if all alerts are ack’ed or have a status other than open.
alerts can be shelved by users if they are a nuisance and are cluttering up the console. they will automatically unshelve after a defined period. usually 2 hours. It’s like a user controlled blackout for a limited time.
does that help?
Nick Satterly
@satterly
Oct 01 2019 21:20 UTC
@ngohoa211 I’ve had a think about how the customer field could be updated for an existing alert and have decided that it isn’t possible to edit the alert directly. This is because it could create a database conflict on the unique constraint that ensures alerts are de-duplicated and correlated correctly.
The only way to safely update the customer for an alert is to send the exact same alert again but with the correct customer and delete the old one.
