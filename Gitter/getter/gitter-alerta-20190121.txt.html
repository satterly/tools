@mala57 the problem is that you are not allowed to write to /alertad.log. I haven't had time to dig very deep into it, but I am assuming there is some configuration option missing and it defaults to wanting to write to the root folder.
docker exec alerta-web bash -c "echo 'LOG_FILE = \"/app/alertad.log\"' >> /app/alertad.conf"
followed by
docker restart alerta-web
should solve your problem
