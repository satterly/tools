Hey Guys!

I am using Alerta 7.4.4 web container with postgres backend

Testing alerts that arrive on Prometheus webhook module
I discovered an issue where the automatically re-opened alerts (due to repeated alert) cannot get closed again.
I get an error "Alert is already in closed status (409)"
I can only delete these alerts.

Re-opened alerts are listed on the dashboard with Open status as expected.
In alert's Details Severity shows the transition from Normal -> Critical as expected.
However, the last record in the History of the alert shows the transition to Closed status.
No record about transitioning to Open Status.

All Prometheus plugin generated alerts exhibit this issue.
I tested Grafana generated alerts and those don't have this issue.

I am running Alerta with debugging enabled and could not find helpful logs for this issue.

Is it a known issue? I'd appreciate any hints and directions where to look!
