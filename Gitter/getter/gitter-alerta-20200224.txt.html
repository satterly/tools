It will actually quite a bit because i need to support custom headers.
My test config for the two servers is …
(venv) nsatterl@macbook alerta % cat ~/.alertad-mlm.conf
DEBUG=False

HMAC_AUTH_CREDENTIALS = [
    {
        'id': 'AKID002',
        'key': 'supersecret2',
        'algorithm': 'sha256'
    }
]

DATABASE_URL='postgres:///mlm'

# Plugins
PLUGINS = ['forwarder']

# northbound interface
X_FORWARDED = 'mid-level-manager'
FWD_DESTINATIONS = [
    ('http://mom.local.alerta.io:8080', 'AKID001', 'supersecret', ['*'])  # forward everything ie. alerts and actions
]
(venv) nsatterl@macbook alerta % cat ~/.alertad-mom.conf
DEBUG=False

HMAC_AUTH_CREDENTIALS = [
    {
        'id': 'AKID001',
        'key': 'supersecret',
        'algorithm': 'sha256'
    }
]

DATABASE_URL='postgres:///mom'

PLUGINS=['forwarder']

# southbound interface
X_FORWARDED = 'manager-of-managers'
FWD_DESTINATIONS = [
    # ('http://localhost:9080', 'AKID002', 'supersecret2', ['actions'])  # only forward all actions ie. not alerts
    ('http://mlm.local.alerta.io:9080', 'AKID002', 'supersecret2', ['*'])  # only forward all actions ie. not alerts
]
those hostnames will work for you. they are just DNS pointers to localhost
if using postgres (like me above) you can create to test databases by simply running …
$ createdb mom
$ createdb mlm
let me know if you need help testing. it was a bit fiddly. ideally this would be a docker setp which you could just launch using docker-compose. Working on that...
