Where I've installed Alerta per this:
https://alerta.readthedocs.io/en/latest/gettingstarted/tutorial-1-deploy-alerta.html#tutorial-1
What is the best way to configure the WebGUI and API to run on a different port from 80? Ideally I'd like the WebGUI to stay on 80 and the /api to run on a different high numbered port, 8089 for example.

1.
I see I'd have to edit this, but isn't clear what to do to change port for webui (leave on 80) and/or api
$ sudo vi /etc/nginx/sites-enabled/default

server {
        listen 80 default_server;
        listen [::]:80 default_server;

        location /api { try_files $uri @api; }
        location @api {
            include uwsgi_params;
            uwsgi_pass unix:/tmp/uwsgi.sock;
            proxy_set_header Host $host:$server_port;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

        location / {
                root /var/www/html;
        }
}
  1. How about web console config.json configuration file to set the endpoint to /api:
    $ sudo vi /var/www/html/config.json
    {"endpoint": "/api"}
  2. this need to be adjusted?
    $ vi $HOME/.alerta.conf
    [DEFAULT]
endpoint = http://localhost/api
  3. I'm ok with CORS since I'd be server from the same fqdn, but different ports, correct?

Thanks!

Nick Satterly
@satterly
Jan 08 20:47 UTC
Precisely because CORS means same fqdn and different ports WILL NOT WORK is why the nginx config serves the web UI on / and the API on /api. If you want the API on a different port you will need to add the web UI base path to the CORS_ORIGINS setting for the API.
If you still want to do this then you need to split out the web UI and API server{} config stanza in the nginx config.
A comprehensive example of what the web UI nginx config would look like can be found here … https://cli.vuejs.org/guide/deployment.html#docker-nginx
And an example of a Flask nginx config here … https://flask.palletsprojects.com/en/1.0.x/deploying/uwsgi/
Nick Satterly
@satterly
Jan 08 20:53 UTC
The web UI config.json will need to have the full URL of the API. a simple relative path /api only works if served from the same domain and port.
Let me know how you go.
