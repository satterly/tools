We are running several clusters behind a firewall. Internet access to these machines is not available. For installing software on these servers we have to manually copy the binaries to the servers and then install.

The pip install command provided in the documentation returned errors as it requires internet connection. By switching to local install method I was able to install Alerta-server and Alerta client.

The pip commands I used

  1. pip install --no-index --no-deps -r /u1/alerta/requirements.txt --find-links=/u1/alerta/alerta-server-5.0.2.tar.gz --target /u1/alerta/

  2. pip install --no-index --no-deps -r /u1/alerta/requirements.txt --find-links=/u1/alerta/alerta-5.0.7.tar.gz --target /u1/alerta/

Quick question about alerta and in particular integration with AlertManager. The normal flow would be Prometheus to AlertManager to alerta, with the operator using the alerta GUI to manage alerts, but with AlertManager responsible for triggering (for example) email alerts. Nick Satterly
@satterly

@plwhite Yes, you understand correctly. Alerta can silence alerts in Alertmanager. One thing though, you can use Alertmanager to send emails or whatever, or you can use Alerta plugins to do this. It’s up to you. Try Alertmanager and if you get stuck you could try Alerta which probably is more configurable but a bit more work to set up for notifications/emails.
