Eduard Kracmar
@Edke
Jun 07 2019 12:36 UTC
@satterly currently in evaluation phase, using original docker images and modifying them so it suits our needs. Our modifications so far:
  • we added redis to requirements as we need mailer and amqp and as that one is written in kombu, we can use lightweight broker like Redis
  • when moving Alerta to production, it will be running in Kube, there to move to proper micro services, monolith image needs to be broken down, currently good for evaluation, but better control when multiple services will be created from it
  • I builded separate lightweight alerta-mailer image, that runs alone and connects to amqp and reports to API
  • also builded lightweight alerta-cli image that can be used in production as Kube Job separately
  • currently on top of this alerta-cli image a cron version was built so we can separately outside of alerta-server image run jobs like housekeeping and heartbeat --alert
  • when moving to production, last task remains, to break frontend image and backend so both are separate, frontend as lightweight nginx image with angular assets and than only alerta backend as uwsgi container
question is, is there any interest in this work and maybe new separate repository with all this would be welcomed ? it may be also part of alerta-docker image, where 2 versions of Dockerfile will resist, one that has focus on monolith aproach, another one for micro services, maybe even with updated helm chart, or separate repositories for each ... so many possibilities, in case this would be appreciated as contribution, we would gladly help
