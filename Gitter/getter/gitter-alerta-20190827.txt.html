Nick Satterly
@satterly
Aug 27 2019 17:38 UTC
@kjetilmjos as far as I can tell, this is a docker-compose issue, not an issue that should have to be solved by every application.
docker need to make it easier to inject config (as a file, or however an app needs it) into a container.
there are a number of “work-arounds” mentioned here https://stackoverflow.com/questions/39388877/adding-files-to-standard-images-using-docker-compose
none of them are ideal. But as I say, i think it’s a docker issue to solve. Or portainer, if that’s meant to be a wrapper service that makes it easy to deploy and manage docker.
Kjetil
@kjetilmjos
Aug 27 2019 19:31 UTC
@satterly there is no issue. It can be done by using volumes or bind mount to a folder. But It's faster to set configuration parameters as envirnment files as you then would not need to mount any external file/folder. As it is now I have to mount the config file to set 2 parameters while the rest can be set as envirnoment variables. Why not open up for the possibility to set all variables from environment variables and config file? Yes, portainer is a popular web based GUI for managing docker containers.
Nick Satterly
@satterly
Aug 27 2019 19:35 UTC
because there are 200 configuration options (see https://github.com/alerta/alerta/blob/master/alerta/settings.py) and some are complex json dictionairies such as ASI_QUERIES and most of the LDAP and SAML options) and some are multi-type lists that can contain numbers or strings or both and some have complex regexs (such as CORS_ORIGINS).
Nick Satterly
@satterly
Aug 27 2019 19:43 UTC
And I don’t want to have to waste time debugging issues people have because their environment variables aren’t being parsed correctly. Because I just know there would be problems. I”d prefer to spend my time adding features.
The only possible way I maybe could possibly be persuaded is if there is a python package that someone has extensive test cases for that is a drop-in replacement.
Even then I’d be very reluctant (for the reasons given above).
And all that to save the bother of writing a 2-line configuration file? If you tell me what the configuration options are, I might be persuaded to add just those two (but I’m not promising anything :)
_
