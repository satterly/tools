@madankumarj In case of mongo I have followed mongo tutorial: https://docs.mongodb.com/manual/tutorial/deploy-replica-set/
For simple replicaSet you have do deploy it on at least 3 machines.
I have deployed a sharded cluster with single replicaset as a member for now, but I'm planning to add another replicaset, and shard alerts collection, by custom localization attribute . For sharded cluster you need:
at least 3 servers for each replica set
at least 3 servers for "config" servers
at least 1 server for mongos router
@madankumarj about emperor - this guide looks reasonable: http://daeyunshin.com/2013/01/06/nginx-uwsgi-django-flask-deployment.html
you need nginx at front and uwsgi running at backend. Each uwsgi emperor vassal listens on tcp port, which is set up as an upstream in the nginx config. I can upload my configs later on if you'd like to.
