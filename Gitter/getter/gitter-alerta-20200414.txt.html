Apr 14 14:27 UTC

Hi, I've been trying the Alerta Rest API to set some ACK actions with timeout. When sending the PUT request, the Alert status is set to ACK status but no timeout seems to be set. I have seen that the Alerta UI sends the PUT request with no timeout set. Is the timeout for ACK actions supported? I expected to have the ACK status reverted when the timeout expires but what I have is an ACK for live until is reverted manually. Is there any configuration that must be set up for this?

Hi, according to documentation yes the action alert endpoint support a timeout. @satterly can you confirm the timeout should work for any action?

Nick Satterly
@satterly
Apr 14 18:13 UTC
Unfortunately this is not currently supported. I haven’t heard of auto-unack’ing ACK’ed alerts before. Is this something you have experience of, or just how you expected Alerta to work (not unjustly)?
moix
@moix
Apr 14 18:17 UTC
oh, that's what I read in http://docs.alerta.io/en/latest/api/reference.html#action-alert , maybe I misunderstood the timeout setting in that curl example, that's why I thought was supported
honestly is something I never thought before neither but one colleague from support team told me that it is useful for them:
1) receive the notification of an alert
2) escalate it to proper team: mailing, ticket, ...
3) ack with a timeout, default timeout depending on the severity the incident
so an ack alert will be again visible for them after the timeout period to go again and chase proper team for resolution
Nick Satterly
@satterly
Apr 14 18:44 UTC
Sounds reasonable. Yes, the docs did make it seem like it should be possible. I’ll add it to the backlog. Thanks for the realworld use-case.
moix
@moix
Apr 14 18:49 UTC
awesome thanks!
by the way I think they started using it because is something available in icinga2: https://icinga.com/docs/icinga2/latest/doc/08-advanced-topics/#acknowledgements (Expiring Acknowledgements)
Nick Satterly
@satterly
Apr 14 20:52 UTC
@moix @frasy Auto unack after timeout will be available in next release alerta/alerta-webui#311
