Nick Satterly
@satterly
May 20 2019 21:56 UTC
@te4336 The API server configuration needs to be in plain text. The uwsgi server (or whatever http server you are using) should not be configured to allow the server config to be compromised. You hide the secrets by setting strict read permissions on the config file and running uwsgi as an non-root user.
This is nothing specific to do with Alerta. Any web application would need to be configured with security in mind. With or without using CI/CD pipelines.
