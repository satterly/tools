When using the delete fucntion in a plugin i'm running into an issue when attempting to federate two alerta instances.

Instance B get an alert from an outside source and fowards it to instance A with an additional attribute of a return
API URL.

The plugin is set so that if that additional attribute exists then it should attempt to remove it from the orginal
instance (B) and if not then assume its being delete on the origin instance and remove it from the central instacne (A)

The code I have creates a loop where If deleting on B then when the plugin on A is followed it tries to remove from B.

# Delete from remote originating Alerta instance before removing locally
        if alert.attributes.get('federation_return_api_url'):
            client = Client(alert.attributes.get('federation_return_api_url'), key=FEDERATION_API_KEY)
            client.delete_alert(id=alert.id)
        # Delete from cental Alerta instance before removing locally
        elif FEDERATION_API_URL or FEDERATION_API_KEY:
            client = Client(endpoint=FEDERATION_API_URL, key=FEDERATION_API_KEY)
            try:
                client.delete_alert(id=alert.id)
            except Exception as e:
                LOG.error('Failed to delete remote alert: %s', e)
                return False
            return True
        else:
             return True

I hitting a mental block on how to get this to work. With the current info I don't see a way to tell instance don't
try to delete from the other instance he is trying to delete from you.

any ideas ??
