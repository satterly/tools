Nick Satterly
@satterly
Jul 16 2019 18:58 UTC
@moix oh yeah sorry
BasicAuth can only be used as the authentication method of you are using the internal auth provider. ie. If using LDAP or SAML or OAuth you must use API key or a Bearer token for webhooks.
Does that make sense?
Nick Satterly
@satterly
Jul 16 2019 19:07 UTC
This is because passwords for external auth providers are not stored by Alerta. This is by design and is very important security measure. It would be grossly irresponsible for Alerta to keep copies of Google or LDAP passwords that may be used to protect business critical data.
The whole point of auth mechanisms like OAuth is to give authorization for something using user credentials without the requester actually having to handle those credentials.
I hope that makes sense and helps you understand why username/password auth cannot be used to authenticate webhooks if an external auth provider is used.
