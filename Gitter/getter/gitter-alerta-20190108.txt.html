@sanderboele glad you got it working although it sounds like it was much harder than it needed to be. my approach would be to split this up into separate docker containers. there’s no reason it all needs to be in one container. and actually, “docker people” say one container for one job.
ie. one container would be Alerta (some have even split Alerta into an API container and a web container when deploying) and separate containers for AMQP and another for the alert mailer.
And yes, you’re right. you should never need to log into a container to configure it. if you have to do that either the container isn’t built right or you’re doing something wrong. You can set most common alerta config variables using environment variables but for more complicated setups you create a alertad.conf file and mount it into the container.
