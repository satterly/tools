@yosefy Only “event” and “resource” are mandatory but the if you have the “reject” plugin enabled (which it is by default) then it will reject alerts that don’t have a valid environment (ie. either “Production” or “Development”) and don’t have something set for “service"
if you don’t want this just set PLUGINS=[] in your alertad.conf server config file.
for a full list of supported statuses see this http://alerta.readthedocs.io/en/latest/api/alert.html?highlight=status#alert-status
For prometheus integration to work correctly the “instance” and “alertname” need to be set by Prometheus at the very least. see this https://github.com/alerta/prometheus-config#configuration---rules
