@alexb145 The search box in the menu bar across the top defines the alert query sent to the API. It uses the same query string syntax as Elasticsearch (if you are familiar with that). it is very powerful as you can target specific alert attributes to search on. however, if you don't specify an attribute name it will default to alert text. See this web page for full syntax http://docs.alerta.io/en/latest/api/query-syntax.html#query-string-syntax
The text search filter menu that pops out from the right does an immediate filter on whatever alerts are currently being displayed. It does not support any query syntax. You just type and it will do a case-insensitive search across every field in all the displayed alerts.
They are two different, but complementary, ways of finding the alerts you are looking for.
Does that help?

Nick Satterly
@satterly
Sep 23 2019 19:02 UTC
@Chandants Use LOG_CONFIG_FILE to set the path to a python logging configuration file and you have total control over logging https://docs.alerta.io/en/latest/configuration.html?highlight=LOG_CONFIG_FILE#config-file-settings
https://docs.python.org/3.7/library/logging.config.html#configuration-dictionary-schema
Nick Satterly
@satterly
Sep 23 2019 19:09 UTC
Define multiple LDAP domains in the Alerta config with each OU you require. 
LDAP_DOMAINS = {
    ‘ou1': 'cn=%s,dc=my-domain,dc=com’,
    ‘ou2': 'cn=%s,dc=my-domain,dc=com’,
    ‘ou2': 'cn=%s,dc=my-domain,dc=com’
}
you could probably think of better names than ou1 though.
