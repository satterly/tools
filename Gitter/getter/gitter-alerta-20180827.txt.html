Nick Satterly
@satterly
Aug 27 2018 16:06 UTC
@kjetilmjos the default docker container does a pip install from Pypi. you’d need to replace that with a COPY or ADD to copy your branch into the container and run pip install -e . or similar. I would guess it should work but can’t be certain without trying myself.
Did you mean to say "Is it possible to not go into any of the alarm lists except if the heartbeat is NOT received?"
if so, then no. “normal” alerts must be received and processed by Alerta to clear down active alerts. If you only send the “non-normal” alerts then they would never clear and you would never know when something is fixed.
you can delete normal alerts almost immediately if you think they are clutering up your “closed” view. just run the housekeeping job every minute or so. Be careful not to DoS your own server though.
