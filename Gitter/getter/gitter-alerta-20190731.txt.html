anttof
@anttof
Jul 31 2019 14:12 UTC
Hi all, quick question. Does anyone know if you can use all possible labels/annotations that alertmanager sends for an alert and use these as columns? I'm currently having some trouble getting this to work and I'm not sure if I understand the "COLUMNS" in settings file correctly
For example: I have the following annotations that I can verify in the "data" tab that alertmanager has sent for an alert:
"annotations": {
"description": xxxxxx high % CPU Memory",
"summary": "CPU Memory Usage: 82%"
But I only see "description" as a column, even though I have added "summary" to the COLUMNS list in settings file
same goes for other labels (not annotations) that I want to be listed as columns
Nick Satterly
@satterly
Jul 31 2019 14:18 UTC
description is an alert attribute that Alertmanager and Alerta just happen to share, and which the webhook just copies straight over for you. Alertmanager labels/tags are mapped as best they can. See https://github.com/alerta/prometheus-config#configuration---rules for list of mappings
Nick Satterly
@satterly
Jul 31 2019 14:24 UTC
Any Alertmanager labels that don’t map to Alerta attributes get copied over as Alerta “tags” and any Alertmanager annotations that don’t map to Alerta attributes get copied over as Alerta "custom attributes”.
anttof
@anttof
Jul 31 2019 14:26 UTC
ah I see, is it possible to use these "tags" or "custom attributes" as columns?
Nick Satterly
@satterly
Jul 31 2019 14:26 UTC
Yes, so if you have labels that you want to see in the alerta console add “tags” to the list of columns.
And if you have annotations you want to see you should just have to add that annotation name to the list of columns eg. “region” or “zone”.
Let me know if that works for you.
anttof
@anttof
Jul 31 2019 14:28 UTC
Ok cool, thanks! I'll try that out
One thought though; Is it possible to add a single specific tags only? For some alerts for example I have 10+ tags, out which maybe only 2 are actually of interest
I could filter that in Alertmanager rules otherwise I suppose
Nick Satterly
@satterly
Jul 31 2019 14:30 UTC
you could convert them into attributes in a plugin pre_receive() hook.
or filter out the unwanted tags in a plugin as well.
anttof
@anttof
Jul 31 2019 14:35 UTC
ah thanks again, I found an example of a pre_receive() plugin online, seemed quite straight forward, I'll give that a try. Now my tags are showing at least
Nick Satterly
@satterly
Jul 31 2019 15:42 UTC
:+1:
