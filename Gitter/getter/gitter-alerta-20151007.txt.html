you can’t serve the API on https:// and expect the web UI to work if it is on http://
they either must BOTH be on https:// or BOTH be on http://
you get a “mixed content” error because they are on different protocols.

the whole point of using nginx to reverse proxy the API onto the /api path is that you are using the SAME ORIGIN for both the API and the web UI so you fix the “mixed content” error and avoid any CORS errors at the same time.
