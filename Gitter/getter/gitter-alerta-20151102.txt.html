@blysik that would work, however you want to be sure it doesn’t block if the mail server is unavailable. you should timeout agressively otherwise alerta migth hang.
just so that you know, there is a mailer daemon that was written a while ago that listens for alerts pushed to an AMQP message bus. the key difference with a simple plugin is that it will hold on to an alert for a short period of time (30 seconds) and if a clear/normal alert is received within that window it will not send an email. we used this for a long time before we switched to pager duty and it worked well for us.
check out the smtp mailer here https://github.com/alerta/alerta-contrib/blob/master/integrations/amqp/mailer.py

by the way, you don’t need to go to the trouble of installing RabbitMQ or whatever just to get an AMQP message queue running. The Kombu python package that alerta uses to connect to AMQP supports lots of different transports. set AMQP_URL to your REDIS_URL (eg 'redis://localhost:6379/') in your alertad.conf file. Do not edit the settings.py in the alerta package directory.
you MUST enable the amqp plugin and then all alerts will automatically be pushed to the AMQP message bus. if the mailer is running it will listen for all alerts on the message bus.
actually, now that i’m looking at it closely i can see that it hasn’t been updated to work with version 4 of alerta.
