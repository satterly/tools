@ANittoor that sounds like a reasonable work flow. the /users API endpoint should provide you with the list of known users, although currently this requires adminpermission. there may need to be a new endpoint to show alerts assigned to a particular user, or the /alerts endpoint with filters might be good enough. that would need to be tested. the web UI changes would be the most involved. It’s more than i could commit to right now though. however, perhaps yourself or someone else would be willing to look at implementing it.
@atya5 there are many ways to integrate Alerta with an alert source. See http://docs.alerta.io/en/latest/integrations.html for some examples. historically the easiest approach is if the custom tool can run a script when an alert is generated then just using the alerta CLI is easiest. Recently the ability to support “custom webhooks” has been added so if the custom tool can call a URL when it generates an alert then that can be a good approach too. alerta/alerta#489

Just some explanation, we are using Alerta in first line with Jira in second line. I have written an plugin that enriches the alerta from our monitoring systems and one plugin that handles communication with Jira.
When the user enters an ticket and clicks on acknowledge they are prompted with an popup where they enter 3 fields that are required by Jira
These fields are added as attributes on the alert in Alerta and then the second plugin sends them to Jira when alert is in status ack.
